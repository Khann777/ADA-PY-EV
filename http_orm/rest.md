API (Aplication Programming Interface) - То, как наше приложение долнжо выглядеть (наш сайт)

Payment API - интерфейс платежной системы

ПРИЛОЖЕНИЕ - ваш сайт, ваш бэк


REST - сводка правил (стандарт) по которому должны писаться API, в REST есть 6 основных правил по тому, как вы должны писать ваши приложения
    -- Отделение клиента от сервера (Client-Server) - Клиент это всегда пользовательский интерфейс сайта/приложения. В REST API код запроса остается на стороне клиента, а код доступа к данным на стороне сервера
    -- Отсутствие состояния клиента (Stateless) - Сервер не должен хранить данные о проведенных клиентом операций
    -- Кэшируемость (Cashable) - Сервер должен небольшой промежуток времени хранить данные о запросе пользователя
    -- Единообразие интерфейса (Uniform Interface) - Все данные должны запрашиваться через один URL адрес, и с использование стандартных протоколов (например: https, ws)
    -- Многоуровневая система (Layered System)
    -- Предоставление кода по запросу (Code on demand) - Сервер должен отправлять клиенту только тот код, который запросил сам клиент
